# Сквозные тесты — сценарии

## Сценарий 1. Создание заказа и отслеживание
**Актор:** Клиент  
**Предусловие:** В системе есть клиент, локации «Москва-Склад» и «СПб-ПВЗ», транспортные связи между ними.  

1. Клиент отправляет запрос `create_order` с грузом 5 кг, 0.1 м³ из «Москва-Склад» в «СПб-ПВЗ».  
2. Система создаёт заказ, рассчитывает маршрут, возвращает UUID и статус `CREATED`.  
3. Клиент вызывает `get_order(uuid)` — видит статус `CREATED`, маршрут, стоимость.  
4. Клиент вызывает `get_tracking(uuid)` — видит одно событие `CREATED`.  

**Ожидаемый результат:** Заказ сохранён, маршрут рассчитан, стоимость > 0, статус `CREATED`.

---

## Сценарий 2. Обновление статуса менеджером
**Актор:** Менеджер  
**Предусловие:** Заказ из сценария 1 существует.  

1. Менеджер вызывает `update_status(uuid, "PROCESSING")`.  
2. Система обновляет статус, добавляет запись в историю.  
3. Менеджер вызывает `update_status(uuid, "IN_TRANSIT")`.  
4. Менеджер вызывает `get_tracking(uuid)` — видит 3 события: CREATED → PROCESSING → IN_TRANSIT.  

**Ожидаемый результат:** Статус `IN_TRANSIT`, история содержит 3 записи в правильном порядке.

---

## Сценарий 3. Недопустимый переход статуса
**Актор:** Менеджер  
**Предусловие:** Заказ в статусе `DELIVERED`.  

1. Менеджер вызывает `update_status(uuid, "CREATED")`.  
2. Система возвращает ошибку `InvalidStatusTransitionError`.  

**Ожидаемый результат:** Статус не изменён, возвращена ошибка.

---

## Сценарий 4. Расчёт маршрута (дешёвый vs быстрый)
**Актор:** Клиент  
**Предусловие:** Граф содержит два пути Москва→СПб: авиа (дорого, быстро) и ж/д (дёшево, долго).  

1. Клиент вызывает `calculate_route` со стратегией «Cheapest».  
2. Система возвращает ж/д маршрут.  
3. Клиент вызывает `calculate_route` со стратегией «Fastest».  
4. Система возвращает авиа маршрут.  

**Ожидаемый результат:** Два разных маршрута; дешёвый дешевле, быстрый быстрее.

---

## Сценарий 5. Маршрут для опасного груза
**Актор:** Клиент  
**Предусловие:** Единственный путь A→B запрещает опасный груз.  

1. Клиент вызывает `calculate_route` с `is_dangerous=True`.  
2. Система возвращает ошибку `RouteNotFoundError`.  

**Ожидаемый результат:** Маршрут не найден, внятное сообщение об ошибке.

---

## Сценарий 6. Полный цикл доставки
**Актор:** Клиент + Менеджер  
**Предусловие:** Заказ создан.  

1. Менеджер последовательно обновляет статус:  
   CREATED → PROCESSING → WAITING_DROP_OFF → IN_TRANSIT → ARRIVED → DELIVERED.  
2. После каждого шага клиент запрашивает `get_tracking(uuid)`.  
3. Количество событий растёт на 1 при каждом обновлении.  

**Ожидаемый результат:** Финальный статус `DELIVERED`, 6 записей в истории.
